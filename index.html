Introduction to Website Deployment with GitHub webpages.

We have several ways to deploy a website to the public internet. In order to explain how to apply this processes we will use GitHub Pages to deploy a website. Github Pages is a service offered by Github Pages which are webpages that are hosted and published through Github.


GitHub Deployment Architecture


    -Are you ready to create your GitHub Account ?

    -Great!! Lets do it!

    Click the next link: https://github.com/ and start to create your GitHub Account.

    -So...Whats the next ?
    Well in order to publish a site by using GitHub Pages, we'll need to create a repository on GitHub.
    What's a GitHub repository?
    A GitHub repository is an online, central storage place where we can store files and all the versions of those files. We'll Use the repository created to store the contents of our website.
    Note: It´s pretty important that your repository name use the GitHubPages naming convention. The same in order to publish it properly. GitHub Naming convention: user-name.github.io Note: The Owner field contain the user-name information which will be repeated on the Repository name in order to follow the GitHub naming convention. Once we have created the repository .github.io , we could see it displayed on the webpages as below:

    Fantastic! Now that we have created our repository we have to understand that there are two ways to upload our site to GitHub.
    • I.- Directly on the Github webpages (Front End)
    • II. By using cmd (Back End)

    On the personal I prefer the Front end option , due to it´s the most practical way, However is good to know both options in case that you could face any issue on the website. So... lets explore the backend option:

    • I.- Back end option to upload a site on GitHub (This option is generally used by ruby and jekyll generation files use)

      • By using Git we will push (upload) the contents of the site´s directory to the new repository.
      To do so, we´ll initialize a Git repository in your sites directory.

      • ⦁ Open the terminal and use cd command to navigate to the sites directory.
      • ⦁ Initialize a Git repository with the git init command.
      • ⦁ Specify to Git where the repository will store your content site(The one you created on GitHub earlier). In order to specify the repository using Git is required to add the remote and label it as the origin.
      • ⦁ The remote is the URL of the repository that will store the site´s contents.
      • ⦁ The Origin is an alias for the remote. The before mention is in order to avoide to having all the time type the lengthy remote URL over and over again, you can simply refer to it as origin later on.

      In the terminal , you can add the remote with the next command:

      git remote add origin https://github.com/your-user-name/your-user-name.github.io.git

      the URL https://github.com/your-user-name/your-user-name.github.io.git Is the remote URL that refers to the repository we have created on GitHub earlier.
      ⦁ In the terminal, add the remote that points to the repository we created earlier

      Example:

      git remote rm origin After specify the remote we will check that it has been added properly with the next command: git remote -v (This command will list all the Git remotes and their corresponding URLS).
      After complete the mentioned steps is required to know exactly which files should be pushed to your repository. In case we would like to push all the site´s content to the repository . We will do the following thing:
      1.- Add all your site content to the staging area by using the git add command.
      2.- Save your changes by usin the commit -m "Save my work" command.
      Great, Now lets go to deploy our site!
      Starting deployment by typing the git push -u origin master command. Important: Remember that on this deploy step the authentification credentials will be requested.

      II.- Front end option to upload a site on GitHub (This option is generally used when we create files by using HTML / CSS /JS languages and editors like Notepad++ or Atoms).
      Go to your GitHub webpage and access your repository github.io.
      -Once you are here you can start by uploading or creating files on your master branch.
      # Important: Everything you start on your master branch and you will commmit will be reflected on your URL. Due to the same it will be the main of the branches.
      Is recommendable to use a gh-pages branch to add the new changes and apply the new version over thoses in order to avoid to affect the master branch and be able to apply the rollbacks.
      After commit any change on our master branch we have published our site on the public Internet.
